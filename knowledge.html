The knowledge bank.

The agent's learnings are not a prompt — they're a version-controlled corpus: 316 atomic instincts, 52 compiled detect→fix patterns, and a 30-entry HyperPod Q&A KB. Every entry is grounded in a real failure, carries a positive and a negative example, and is bounded by where it applies. Below is the shape of that knowledge — counts and representative samples, with all customer/account/host specifics excluded.

Coverage, not count. Every instinct is retrievable (reachable by phase or layer), actionable (has a positive and a negative example), and attributed (has a source). That 100/100/100 floor is enforced by a test and a pre-commit hook and never lowers — raw count is treated as a vanity metric.

Representative learnings

Six instincts, one per layer

A sample of high-confidence instincts, customer/account/host specifics removed. Each pairs a rule with a ≤48-char negative prime (“don't do X”) that primes the agent before it acts.

K0003 · L6 os/infra · conf 0.99

On HyperPod EKS p5/p5en, kernel modules (efa_nv_peermem, gdrdrv, nvidia_peermem) can't be loaded from kubectl debug ("Operation not permitted") — use a hostPID DaemonSet.

✗ kubectl debug cannot modprobe; use hostPID DaemonSet

K0005 · L5 efa/fabric · conf 0.99

RDMA fabrics in a cloud VPC silently drop frames on a secondary/overlay CIDR (verified on EFA SRD; same trap on RoCEv2). Use the primary CIDR + a self-referencing SG rule.

✗ SRD drops on secondary CIDR; primary CIDR + self-ref SG

K0011 · L4.5 ci/build · conf 0.99

The aws-efa-k8s-device-plugin v0.5.4 nodeAffinity allowlist omits p6-b200.48xlarge → DaemonSet desired=0 → no node advertises vpc.amazonaws.com/efa.

✗ device-plugin v0.5.4 missing p6-b200 in allowlist

K0060 · L2 framework · conf 0.98

The NIXL_BACKEND family of env vars is not read anywhere in the vLLM connector — setting them is a no-op. Configure via the connector extra-config instead.

✗ NIXL_BACKEND env-var unread; use kv_connector_extra

K0020 · L0 kernel · conf 0.97

A B200 cudaErrorNvlinkUncorrectable is a hardware fault, not a recipe bug — a pod restart re-rolls the dice; cordon the node instead of chasing the software.

✗ NVLink uncorrectable = hardware; cordon, don't re-debug

K0102 · L7 benchmark · conf 0.97

A chat-completion 200 with content but transfer_count=0 and symmetric tx=rx on one NIC is EFA-NOT-VERIFIED — never claim disagg-over-EFA without hardware-counter proof.

✗ completion 200 ≠ disagg verified; prove with hw counters
