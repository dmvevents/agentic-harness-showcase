how we know it works · updated 2026-06-26

Validation by replay,
not by vibes.

We don't claim the agent is good — we measure it. We replay thousands of real historical sessions through the failure classifier (without running a single command), measure how many real failures it catches, and hold every detection pattern behind an adversarial precision gate that keeps false-positives at zero. Disagreement is a feature: where the system diverges from itself, we mine the gap and fix it.

Coverage is a range, stated honestly. On distinct deduplicated failure signatures we measure ~74%; on raw per-line error output (which repeats the same failure many times) it's lower. We report the denominator, never cherry-pick the flattering number.

The method

Replay, don't re-run

The insight: a real session already recorded what went wrong. Instead of re-running commands on a cluster, we pass the recorded failure output through our tool and ask — would it have caught this?

The discipline that matters

An adversarial precision gate

Coverage is easy to fake — match everything and you "catch" 100%. So every pattern is held to a hard rule: a benign or success line must never classify as a failure. A false hit would burn an agent turn chasing a problem that isn't there.

must NOT fire 
# benign / success — 0 may classify
"All pods Running 1/1, healthy"
"busbw 379 GB/s"
"iam:PassRole granted; attached"
"vCPU quota: 1152 of 1152 available"
"exit code 0"
must fire (single deterministic pick) 
# real failures — each classifies
"NET/Socket : busbw 11 GB/s"   // TCP fallback
"Operation not permitted (insmod efa_nv_peermem)"
"version `FABRIC_1.9' not found"
"command terminated with exit code 124"
The 52 patterns we added this round passed three independent gates: 0 compile errors, 0 false-positives on the benign guard, and 0 false-positives on 107 real success lines harvested from sessions — and every one fires on its own real error example. Coverage went 70% → 74% without loosening precision.

Disagreement is a feature

Using divergence to fix the system

The back-test surfaces three kinds of divergence — each is a signal we feed back to improve the agent itself.

DivergenceWhat it revealsThe fix it drives
Tool vs realitya real failure our corpus doesn't classify (a coverage gap)mine it into a new detection pattern (this round: +52)
Pattern vs patternone log matches multiple patterns with conflicting severity (dispatch ambiguity)a deterministic severity + id tie-break picks one fix — never a coin-flip turn
Claimed vs captureda session says "found the root cause" but it never entered the corpusmine the claimed fix into an instinct so the next run starts knowing it
The loop: replay sessions → measure per-layer coverage → find divergences → draft patterns → adversarially verify against the precision gate → merge only what passes → re-measure the lift. It ran end-to-end this round and is repeatable on the remaining gap.
