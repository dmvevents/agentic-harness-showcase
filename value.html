The engine room

The learning machinery, in plain terms

cost control

UCB bandit — spend where it pays

A multi-armed bandit where each arm is an LLM model. It balances exploiting the model that has worked against exploring a cheaper one, using a UCB1 score with asymmetric exploration and exponential decay. The practical effect: the agent doesn't reflexively reach for the most expensive model — it learns which model is worth it for which job.

don't-repeat-mistakes

Meta-learner — a corpus, not a vibe

316 atomic instincts, each with a positive and a negative example and an applies_to qualifier. The negative example is the load-bearing part — it primes "don't do X," which is what stops the agent repeating a known failure. Retrieval filters to the few instincts that actually apply to the task in front of it.

it gets smarter

Log-learner — knowledge from what broke

Real session, deploy, and system logs are mined into candidate instincts through a review-gated pump. The corpus grows from what actually failed in production, not from guesses — and a contributor can't self-approve their own entry. Tribal knowledge becomes a retrievable asset.

positive + negative

Why both examples matter

A positive example shows the fix that worked; the negative example shows the trap that didn't. Together they're what let the agent reduce wasted turns — it recognizes a failure signature and reaches for the codified fix instead of re-deriving it, which is where both the time and the cost savings come from.

What you can run through it

Test · train · infer · optimize

All of these run as catalog modules through the same plan → ladder loop — so each gets the same failure-classification, self-healing, and audit trail.

Test

Unprivileged NCCL allreduce + hardware-counter checks to prove the fabric is real RDMA, not silent TCP.

Train

FSDP, Megatron-LM (TP/PP/DP), and NeMo RL training runs on a validated multi-node cluster.

Infer

Disaggregated inference deploys; latency / throughput measured against an SLA gate.

Optimize

Kernel optimization at L0 via the evolution engine — compile, verify, benchmark, keep the best.

The same discipline applies to all four. A throughput number captured while NCCL silently fell back to TCP isn't slow — it's invalid. The classifier flags that before the number reaches a report. That is the difference between a benchmark and a guess.
