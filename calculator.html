GPU cluster sizing & cost calculator

A pure, client-side calculator: same inputs always produce the same output, with no backend and no network calls. This is the public embodiment of the "deterministic tool" pattern.

Directional, not a benchmark. Every number on this page is a directional, public-spec estimate computed from published vendor specs and public list prices. It is not a measured benchmark, a quote, or negotiated pricing. Real benchmark numbers live in a PR-gated repo with full provenance and a validation status — none of those appear here.

How the math works

Every formula is public and labeled directional

No hidden model, no fitted curve from private data. These are coarse public heuristics, written out so you can audit them in assets/calculator.js (all pure functions).

OutputDirectional formula (public heuristic)
Total GPUs instances × 8 (P-family nodes have 8 GPUs each)
Aggregate VRAM total_GPUs × vram_per_gpu (H100 = 80 GB, H200 = 141 GB, B200 = 192 GB)
Model footprint (inference) params_B × bytes_per_param × 1.25 — weights + a flat 25% KV-cache/activation working set
Model footprint (train / fine-tune) params_B × 16 — the classic ~16 bytes/param Adam mixed-precision rule of thumb
Memory-fit verdict compare footprint to aggregate_VRAM × 0.85 (a usable-fraction haircut for fragmentation/overhead)
Parallelism (TP/PP/DP) TP up to 8 (one NVLink domain) · PP across nodes only if one node can't hold the footprint · DP fills the rest
On-demand cost instance_list_price × instances — public list price, excludes storage / egress / discounts
